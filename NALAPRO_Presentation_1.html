The 20 Newsgroups Dataset

Source & Loading fetch_20newsgroups(subset='train'/'test')
  • 18,000+ newsgroup posts across 20 topics
  • Noise removed: headers, footers, quotes
  • Official sklearn train / test split used
  • All 20 classes → 6 selected for speed
6 Categories Used
sci.space sci.med talk.politics.guns rec.sport.hockey comp.graphics alt.atheism

Covers diverse semantics: science, politics, sports, technology & religion — ideal for testing how well models separate very different topics.

Why remove headers/footers? Email headers often contain the sender's newsgroup address, which directly reveals the category label. Removing them forces models to learn from actual post content, not metadata.

Why fp16?

16-bit floats use half the GPU memory and run ~2× faster with negligible accuracy loss — essential for Colab's free T4 GPU.

Why max_length=256?

BERT max is 512. Using 256 halves memory and doubles batch throughput while preserving most of each post's content.

08 · Discussion

Discussion & Conclusion

More W2V epochs = better embeddings. UMAP shows tighter, more structured clusters with 10 epochs vs 1 — direct visual evidence that repeated exposure improves word representations.
TF-IDF beats Word2Vec despite being simpler. 5000 bigram features capture topic-discriminative rare words (e.g. "space shuttle", "ice hockey") that document averaging in W2V dilutes away.
Hybrid W2V+TF-IDF is the best classical method. Concatenating normalized semantic + statistical features gives the SimpleNN richer signal than either alone — hypothesis confirmed.
BERT dominates all classical methods. Contextual attention across the full document captures meaning no fixed-vector method can match — jump from ~78% to ~91%.
MLM pre-training gives consistent improvement. Teaching BERT newsgroup vocabulary before classification adds ~2% — more impactful on specialized domains with unique vocabulary.
Few-shot beats zero-shot, but both lag fine-tuned BERT. In-context examples help LLM pattern-match the task, but without weight updates, a generalist model can't match a fine-tuned specialist.
Key Trade-off: BERT fine-tuned → best accuracy, needs GPU + labeled data. LLMs → flexible, zero training cost, works without labels. MLM bridges them — unsupervised adaptation before supervised fine-tuning.
